Fix Usage Tracking for All Providers + Per-Thread Usage Stats

TL;DR

Cost/usage tracking is broken for every provider except Claude. Three independent gaps in the ingestion pipeline make Codex, Grok, Cursor, and OpenCode turns record $0.00 cost and 0 tokens. The per-thread usage stats and settings page enhancements the user wants are customers of this pipeline — building them on a broken pipeline would produce misleading zero-data displays. Fix the pipeline first, then add the UI features.

Gap 3 — Model name extraction from modelUsage is structurally wrong

File: apps/server/src/orchestration/Layers/ProviderRuntimeIngestion.ts:1625-1628

const model =
  typeof event.payload.modelUsage?.model === "string"
    ? event.payload.modelUsage.model
    : undefined;

modelUsage is typed as Record<string, ModelUsage> where keys are model names (e.g., { "claude-sonnet-4-6": { inputTokens: ..., outputTokens: ..., costUSD: ... } }). There is no .model property. The correct extraction is Object.keys(event.payload.modelUsage)[0].

Consequence: model is always undefined, so computeCost(model, ...) at line 1635 is never reached. Cost is 0 for any provider that doesn't report totalCostUsd.

Gap 4 — No session-model fallback when turn.completed lacks model info

Every adapter knows the active model at the session level. The session object (ctx.session.model) has the model string. But the ingestion code never falls back to the session model when the turn.completed event doesn't carry it.

Gap 5 — No model alias normalization before pricing lookup

The normalizeModelSlug function exists at packages/shared/src/model.ts:235 and the alias table MODEL_SLUG_ALIASES_BY_PROVIDER exists at packages/contracts/src/model.ts:157. However the ingestion pipeline (which lives in apps/server/src/orchestration/, not in provider code) never calls normalizeModelSlug. A model string like "sonnet" or "5.4" would fail to match PRICING_BY_MODEL.

Additionally, the alias table has no entries for GROK_DRIVER_KIND or OPENCODE_DRIVER_KIND — these need entries added.

Recommendation

Fix the pipeline first, then build the UI on top.

The original request (per-thread usage stats + settings page) is a customer of the cost pipeline. Building it before fixing the pipeline would produce a UI that shows $0.00 for 4 out of 5 providers — worse than no feature at all (users would think the app is misreporting).

Split into two phases:

  • Phase 1 (P0): Fix the 5 ingestion gaps so all providers log correct cost/tokens/model.
  • Phase 2 (P1): Add per-thread usage chip + settings page thread drill-down.

Scope

Phase 1 — Fix pipeline (in scope)

  1. Fix field name mismatch in the thread.token-usage.updated fallback — check both last* and non-prefixed fields.
  2. Fix model name extraction from modelUsage — use Object.keys() to get the model name from the record keys.
  3. Add session model fallback — when turn.completed lacks model info, fall back to the thread's session model.
  4. Apply normalizeModelSlug before pricing lookup — call it with the correct provider driver kind.
  5. Populate turn.completed.payload.usage in GrokAdapter, CursorAdapter, CodexAdapter, and OpenCodeAdapter.
  6. Add thread.token-usage.updated emission to GrokAdapter and CursorAdapter.
  7. Add GROK_DRIVER_KIND and OPENCODE_DRIVER_KIND entries to MODEL_SLUG_ALIASES_BY_PROVIDER.
  8. Fix OpenCodeAdapter to set both inputTokens and lastInputTokens (or fix the fallback to read both).

Phase 2 — UI features (in scope)

  1. Add aggregateByThreadId + aggregateByAllThreads to TurnCostRepository.
  2. Wire threadId filtering in getUsageSummary RPC handler.
  3. Add getUsageSummaryByThread RPC.
  4. Client: extract shared useUsageAggregate hook + display components.
  5. Client: add ThreadUsageChip in ChatHeader (per-thread cost/tokens).
  6. Client: enhance UsageSettingsPanel with thread-level breakdown.

Out of scope (for now)

  • Sidebar per-thread cost badges (needs schema + projection change).
  • Per-turn cost on fold rows.
  • Date-range filtering.
  • Export/download usage data.
  • Real-time usage during running turns.

Implementation Plan

Phase 1 — Fix pipeline for all providers

Fix 1: Fix the thread.token-usage.updated fallback to read both last* and non-prefixed fields

File: apps/server/src/orchestration/Layers/ProviderRuntimeIngestion.ts:1616-1621

Change from:

inputTokens = cachedUsage.lastInputTokens ?? 0;
outputTokens = cachedUsage.lastOutputTokens ?? 0;
cachedInputTokens = cachedUsage.lastCachedInputTokens ?? 0;
reasoningTokens = cachedUsage.lastReasoningOutputTokens ?? 0;

To:

inputTokens = cachedUsage.lastInputTokens ?? cachedUsage.inputTokens ?? 0;
outputTokens = cachedUsage.lastOutputTokens ?? cachedUsage.outputTokens ?? 0;
cachedInputTokens = cachedUsage.lastCachedInputTokens ?? cachedUsage.cachedInputTokens ?? 0;
reasoningTokens = cachedUsage.lastReasoningOutputTokens ?? cachedUsage.reasoningOutputTokens ?? 0;

This fixes OpenCodeAdapter (which sets inputTokens not lastInputTokens).

Fix 2: Fix model name extraction from modelUsage

File: apps/server/src/orchestration/Layers/ProviderRuntimeIngestion.ts:1625-1628

Change from:

const model =
  typeof event.payload.modelUsage?.model === "string"
    ? event.payload.modelUsage.model
    : undefined;

To:

const modelUsageRecord = event.payload.modelUsage;
const rawModel =
  modelUsageRecord && typeof modelUsageRecord === "object"
    ? (Object.keys(modelUsageRecord).find((k) => typeof k === "string") ?? undefined)
    : undefined;

modelUsage is Record<string, ModelUsage> where keys are model names like "claude-sonnet-4-6" (this is how the Claude SDK structures it).

Fix 3: Add session model fallback

File: apps/server/src/orchestration/Layers/ProviderRuntimeIngestion.ts

When rawModel is still undefined after Fix 2, try to get the model from the thread's session. The session model is available through the existing ProviderService or the thread's session data. Pass the provider instance ID through and look up the configured model.

The simplest approach: add a Map<ThreadId, string> that tracks the active model per thread from session start events. session.started events carry a model field. Cache it at the same scope as latestTokenUsageByThread.

Fix 4: Apply normalizeModelSlug before pricing lookup

File: apps/server/src/orchestration/Layers/ProviderRuntimeIngestion.ts (around line 1634)

import { normalizeModelSlug } from "@t3tools/shared/model";

// ... after model extraction:
const normalizedModel = model ? normalizeModelSlug(model, event.provider) : null;
let costUsd: number | undefined;
if (typeof event.payload.totalCostUsd === "number") {

} else if (normalizedModel) {

Also store the normalized model in the DB row instead of the raw model (or store both — store the normalized form for cost attribution).

Fix 6: Populate turn.completed.payload.usage in all adapters

For each non-Claude adapter, extract token counts from the provider's turn completion data and populate payload.usage = { input_tokens, output_tokens, cached_input_tokens }.

  • CodexAdapter (CodexAdapter.ts:808): The thread/tokenUsage/updated event carries token counts in last.inputTokens, last.outputTokens. Read from the most recent cached usage at turn.completed time.
  • GrokAdapter (GrokAdapter.ts:868): If the turn completion carries usage, map it.
  • CursorAdapter (CursorAdapter.ts:1043): Similar — check if the completion data has token counts.
  • OpenCodeAdapter (OpenCodeAdapter.ts:916, 945, 1259): The step completion data has input and output token counts. Map these to { input_tokens, output_tokens }.

This is the most adapter-specific fix. Each adapter may need different extraction logic. The important thing is to set payload.usage so the ingestion code finds tokens directly in turn.completed rather than relying on the fallback.

Fix 7: Emit thread.token-usage.updated from GrokAdapter and CursorAdapter

If the adapter doesn't have direct per-turn usage data at turn-completion time, at minimum emit periodic thread.token-usage.updated events so the fallback has data to work with.

GrokAdapter: Add emission of thread.token-usage.updated when context window updates are available. Follow the pattern from CodexAdapter (CodexAdapter.ts:768-774) or OpenCodeAdapter (OpenCodeAdapter.ts:1013-1025).

CursorAdapter: Same — emit thread.token-usage.updated events.

Fix 8: Fix OpenCodeAdapter to set last* fields too, or ensure Fix 1 handles it

Fix 1 already handles the case where lastInputTokens is undefined by falling back to inputTokens. But for consistency and to avoid relying on fallback ordering, also add lastInputTokens / lastOutputTokens to the OpenCodeAdapter's thread.token-usage.updated payload. File: OpenCodeAdapter.ts:1015-1016.

Risks & Notes

  • Phase 1 is the prerequisite. Don't start Phase 2 until Phase 1 is verified with real data.
  • Claude must not regress. Claude is currently the only provider with correct cost tracking (because the SDK reports total_cost_usd). Fixes to model extraction must not break this.
  • Session model fallback is approximate. If a user changes the model mid-thread, the session model at turn-completed time may differ from the model used for the turn. This is a minor edge case. For precise attribution, each adapter should report the per-turn model in turn.completed.
  • OpenCode's cost field: OpenCode's step completion data includes a cost field (OpenCodeAdapter.ts:1009). This could be mapped to payload.totalCostUsd for direct cost tracking without pricing table lookups. Investigate this as an additional improvement.
  • readLocalApi().server.* only works in Electron. The web mode uses WebSocket RPC via connection.client.server.*. The shared hook needs to handle both paths.
